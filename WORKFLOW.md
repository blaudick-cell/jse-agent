# Workflow

1. Read CURRENT_OBJECTIVE.md, PROJECT_CONTEXT.md, KNOWN_FAILURES.md, and NEXT_ACTIONS.md first.
2. Plan before acting.
3. Execute one small task at a time.
4. Verify every result.
5. Update markdown memory after meaningful progress.
6. Run ./checkpoint.sh after meaningful file changes.
7. Never retry the same failed action more than twice.
8. If blocked, stop, explain blocker, and propose workaround.
